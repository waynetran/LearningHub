# LearningHub

A field guide to what is actually happening at the edge of math, science, and technology — and deep dives for when you want to know why.

LearningHub is a static reference site for tech professionals who want to catch up on the math, CS, and AI/ML foundations behind modern computing without being talked down to, and without being assumed to remember everything. Every page defines its symbols, works through examples, ships an interactive figure, and cross-links its prerequisites.

## Tracks

- **Mathematics** — calculus, multivariable calculus, linear algebra, probability, statistics, discrete math, numerical analysis
- **Computer Science** — algorithms & data structures, complexity & Big-O, theoretical CS, mathematical optimization
- **AI / Machine Learning** — a historical tour from the 1957 perceptron through today's reasoning models, with deep dives on perceptrons, backprop, CNNs, RNNs, attention, embeddings, GANs, diffusion, foundation models, RAG, mixture-of-experts, state space models, test-time reasoning, edge AI, and alignment

Each of the three tracks has a sidebar-nav "track hub" (`math/index.html`, `cs/index.html`, `ai-ml.html`) that summarizes the field inline with interactive demos, plus deep-dive pages for every subject and topic.

## How it's built

Pure HTML, CSS, and vanilla JS — no build step, no framework, no server needed. Every page works from `file://`. Interactive figures are inline SVG with small IIFEs. Math rendering uses KaTeX loaded from a CDN (subject pages only). Site-wide search uses a precomputed static JSON index generated by `build-search-index.py`.

```
index.html              ← landing page
ai-ml.html              ← AI/ML track hub
math/index.html         ← Mathematics track hub
cs/index.html           ← Computer Science track hub
topics/*.html           ← AI/ML deep dives
math/*.html             ← Mathematics subject pages
cs/*.html               ← Computer Science subject pages
styles.css              ← all styling (single file)
search.js               ← client-side search UI
search-index.js         ← precomputed search index (rebuild with build-search-index.py)
build-search-index.py   ← scans all HTML and regenerates the search index
CLAUDE.md               ← design contract — read before editing
```

To rebuild the search index after editing any page:

```sh
python3 build-search-index.py
```

To serve locally (optional — the site works on `file://` too):

```sh
python3 -m http.server 8000
```

Then open <http://localhost:8000>.

## AI-generated content disclaimer

**This entire site — every page of prose, every formula, every interactive demo, every diagram, and most of the code — was generated by [Claude Code](https://claude.com/claude-code) using Anthropic's Claude Opus 4.6 model.** A human directed the effort, shaped the structure, and reviewed the output, but the generative work was done by an AI system.

Large language models are confident and frequently wrong. Treat this site as a study aid and a starting point, not as a primary reference. If you are using it to learn something important — for an interview, a paper, a product decision — verify anything load-bearing against a textbook, a peer-reviewed source, or working code you can run.

Known categories of potential error:

- **Dates and attributions** — the year a paper was published, who authored it, which lab released which model. These are the kinds of facts LLMs hallucinate most often.
- **Formula details** — a transposed index, a missing constant, a wrong sign. The big-picture derivations are usually right; the bookkeeping sometimes isn't.
- **Historical claims** — "first to do X", "state of the art as of Y", "only N labs are doing Z". Check before citing.
- **Benchmark numbers** — specific percentages, leaderboard ranks, parameter counts. Often out of date or off by a factor.

If you find something wrong, please tell us — see below.

## Contributing

Contributions are welcome and encouraged. If you spot a mistake, a typo, an outdated date, a broken link, a formula error, or a better way to explain something:

- **File an issue** describing what you found: <https://github.com/waynetran/LearningHub/issues>
- **Open a pull request** with the fix: <https://github.com/waynetran/LearningHub/pulls>

Small fixes are especially welcome — correcting a date or a formula takes seconds and helps every future reader. Larger contributions (a new subject page, a new interactive demo, a rewrite of a confusing section) are also welcome; please open an issue first so we can agree on scope.

When editing, read `CLAUDE.md` — it's the design contract that every page is supposed to follow (math-glossary rules, prereq badges, layered depth, cross-linking, voice and tone).
